## Non-negotiable constraints

1. **Local-first (ADR-001).** The shipped distiller and readiness check run on-device.
   No user corpus leaves the machine. Cloud Opus is **offline validation / training-data
   only** and, at most, an optional consent-gated toggle (default off).
2. **Provenance on every unit** â€” each answer-unit links to its source section; surfaced
   as "expand to source" in the live card. Never show un-attributable text.
3. **No overfitting** â€” the distiller extracts generally; the readiness question set is
   held separate from the distiller. Coverage must reflect a real corpus, not a gamed one.
4. **Gates:** `ruff check .`, `mypy --strict` on `lib/`/`src/`, `pytest` all green before
   each commit. New pure logic gets colocated tests in `tests/`. Files < 300 lines.
5. **Workflow:** branch `liquid-rearch`; commit per task; messages in the developer's
   voice, no "Co-Authored-By"/"Generated by". Never push or touch `main` without a green
   light.
